# 风里骑局：即时配送政策推演的数字博弈系统

## 项目定位

本仓库是数字经济学课程的结课作业，目的仅在于把课堂里的多方博弈模型做成一个可调参的演示 Demo。所有参数、图表与政策建议都为教学示例，不具备真实业务或政策参考价值，也没有任何商业化计划。

## 背景与目标

- 用最直观的方式展示“调佣金”“降劳动强度”等策略会如何影响平台利润、骑手效用、社会福利；
- 把课堂里的平台、骑手、消费者三方讨论整合进单页仪表盘，方便课堂讲解；
- 提供一键导出报告/数据的功能，便于完成课程汇报或附录。

## 核心模块

1. **参数面板**：滑块控制佣金率、劳动强度、算法效率、等待容忍度、社会平衡系数以及监管、竞争、监控、创新等环境变量。
2. **推演与提示条**：点击“开始推演”后，以最新参数刷新 KPI 卡片、演化曲线、雷达图、可持续度条和算法压力条。若调整参数后未推演，提示条会提醒。
3. **情景预设**：内置“基准 / 强监管 / 激烈竞争 / 技术突破”四个组合按钮，方便课堂演示不同策略。
4. **底部分析面板**：包含成本效益环形图、市场结构卡（集中度、创新指数、监管有效性）以及粒子式订单流显示。
5. **导出功能**：报告导出 Markdown（含执行摘要、参数、主要指标和建议），数据导出 CSV（记录时间戳、所有参数、需求、利润、效用、效率等指标）。

## 使用方法

1. 推荐在仓库根目录运行 `npm install && npm run start` 或执行 `start.sh / start.bat`，开启本地静态服务器；也可以直接双击 `index.html` 打开，但部分导出功能在 `file://` 协议下可能受限。
2. 打开页面后先通过左侧滑块或情景按钮设置参数。
3. 点击右上角“开始推演”，所有图表与 KPI 会同步刷新。
4. 若需要课堂材料或报告，点击“生成报告”；需要原始数据则点击“导出数据”。
5. 所有结果仅用于课程讨论，若引用需注明为课堂作业示例。

```

├── index.html        # 页面结构与布局
├── styles.css        # Tailwind 自定义样式

├── model.js          # 核心经济模型与指标计算

├── utils.js          # 报告/CSV 导出、格式化工具
├── start.sh / start.bat / package.json  # 本地服务脚本
└── 模型公式说明.md     # 公式与假设详解
```

## 公式与假设

模型中涉及的需求函数、平台利润、骑手效用、消费者剩余、社会福利以及高级指标的推导，都记录在《模型公式说明.md》。该文件与 `model.js` 互相对照，便于在课堂报告或代码维护时快速查阅。
