# 风里骑局模型公式说明

1. 需求函数

---

需求随价格、等待时间、演算法效率和系统容量变化：

```
price_factor   = (1 - r × 价格敏感度)^α
wait_factor    = (τ / T)^质量偏好
network_effect = 1 + 忠诚度 × ln(1 + η_调整)
capacity_factor = clamp((η_调整 × e_有效) / 3)
D = A × price_factor × wait_factor × network_effect × competition_factor × macro_factor × capacity_factor
```

2. 平台利润

---

在订单量驱动下计算单位骑手订单数、分成收入及多项成本：

```
orders_per_rider = (η_调整 × D) / 骑手数
base_income = (1 - r) × P_order × orders_per_rider × 分成系数
P = revenue + data_revenue - variable_costs - fixed_costs - tax - employer_social_cost
```

3. 骑手效用

---

将劳动压力、监控、算法偏见等因素转化为成本，叠加保障收益：

```
p_base = max(0, (T - τ)/τ)
p_monitoring = 监控强度 × 系数
p_bias = 偏见系数 × (1 - 监管 × 0.5)
体力/压力/培训/安全成本 = 函数
Ur_single = base_income + 社保 + 最低工资补差 - 成本
Ur = Ur_single × 骑手数
```

4. 消费者剩余与社会福利

---

消费者体验、外部性及补贴共同决定 welfare：

```
CS = max(0, (价格剩余 + 等待奖励 - 体验惩罚) × (D/100))
总外部性 = 环境 + 拥堵 + 不平等 + 算法偏差 + 财政补贴 - 就业 - 创新溢出 - 数字化收益
SW = P + Ur + CS - 总外部性
```

5. 高级指标

---

用于 KPI 卡片与雷达图的组合指标：

```
市场效率 = (SW / (P + Ur + CS)) × 100%
收益差距指数（基尼）= 基于平台/骑手/消费者的洛伦兹曲线计算
可持续指数 = max(0, min(100, 100 - (环境成本 / |SW|) × 100))
创新指数 = 创新水平 × η_调整 × 100
监管有效性 = 监管强度 × (SW / 20000) × 100
```

> 所有公式皆为教学用途的简化版本，重在展示相对趋势而非精确计量结果。
